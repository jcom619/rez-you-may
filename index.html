A CSS and JavaScript based (“dynamic variable”  responsive) styling library that changes the CSS values of an element according to what the weather conditions and time of day are in a given local area

Very simple to to reference library by applying script tags and link tags to the appropriate file types accompanied by the appropriate attributes and selectors to keep things simple with a universal range of compatibility. Basically a CDN using vanilla JavaScript and html with Node.js for a few pertinent middleware dependencies.

 (express, dotenv, bcrypt and axios )

  • Styles are easily applied by inserting the pre-defined CSS “class names” to the desired elements class attribute.

  • All predefined styles can be redefined by you as a developer in local CSS files.

  • Variable stipulations make reference to a 3rd party API ( https://openweathermap.org ) that communicates local weather conditions to determine the dynamic outcome of styles.
  • These rules are statically applied, for now...but I will eventually write in the option to re-apply what variables are being checked or just calibrate what is currently referenced. This is far down the priorities list I have set for this project so expect many other more useful updates before this happens such as automatic geolocation, completing the style library and generating as precompiled SASS/SCSS to optimize and “minify” payload or packaging as a node module with an NPM install option..

  •  I built this with HTML/CSS/JS,ES6 and Node ( for middlewares and encryptions ) to get a more universal compatibility and to simplify troubleshooting.

  • The functionality and methods are very straightforward in this specific scenario and I wanted to plan this one so it would be able to evolve greatly as the necessities and/or opportunities presented themselves ( as they tend to do ), I ultimately feel that if I present this under the guidelines of a 3rd party framework I sacrifice scalability range and add complexity that would serve only as more code to write and more places to write it for no added benefit.
